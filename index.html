🐳 DOCKER • NGINX • APACHE
Simple Study Guide — No Jargon — Just Clear Examples
🐳 DOCKER — What Is It?
📦
What is Docker?
A container is like a portable box that holds your app + everything it needs to run. It works the same on any computer.
📋
Image vs Container
Image = Blueprint / Recipe
Container = A running copy of that blueprint
📁
Dockerfile
A recipe file that tells Docker how to build your image. You write it, Docker follows it.
📝 Example Dockerfile: FROM nginx:alpine # Start with a lightweight web server COPY index.html /usr/share/nginx/html/ # Copy your file EXPOSE 80 # Tell Docker to use port 80 CMD ["nginx", "-g", "daemon off;"] # Start the web server
⚡ Common Docker Commands: docker build -t my-app . # Build an image from Dockerfile docker run -d -p 8080:80 my-app # Run a container, map port 8080→80 docker ps # List running containers docker stop my-container # Stop a running container docker rm my-container # Remove a stopped container docker images # List all images docker rmi my-image # Remove an image
🦈 NGINX — What Is It?
🌐
What is Nginx?
A web server that delivers your website files to visitors. It's fast, lightweight, and handles many connections at once.
⚡
Why Use Nginx?
Fast — handles thousands of connections
Lightweight — uses very little memory
Reverse Proxy — can route traffic to other services
📁
Common Uses
• Serving static websites (HTML, CSS, JS)
• Load balancing
• SSL/TLS termination (HTTPS)
• Reverse proxy for APIs
📝 Simple Nginx Config (nginx.conf): server { listen 80; server_name mysite.com; root /var/www/html; index index.html; }
⚡ Common Nginx Commands: sudo systemctl start nginx # Start Nginx sudo systemctl stop nginx # Stop Nginx sudo systemctl restart nginx # Restart Nginx sudo nginx -t # Test configuration sudo nginx -s reload # Reload without restarting sudo tail -f /var/log/nginx/access.log # View access logs
🦁 APACHE — What Is It?
🌐
What is Apache?
The most popular web server. It's been around since 1995 and powers millions of websites. Like Nginx, it delivers web pages to visitors.
🔄
Nginx vs Apache
Apache = Older, more features, uses more memory
Nginx = Newer, faster, uses less memory
📁
Common Uses
• PHP hosting (WordPress)
• .htaccess support
• CGI / Perl scripts
• Legacy applications
📝 Simple Apache Config (000-default.conf): <VirtualHost *:80> ServerName mysite.com DocumentRoot /var/www/html <Directory /var/www/html> Options Indexes FollowSymLinks AllowOverride All </Directory> </VirtualHost>
⚡ Common Apache Commands: sudo systemctl start apache2 # Start Apache sudo systemctl stop apache2 # Stop Apache sudo systemctl restart apache2 # Restart Apache sudo apache2ctl configtest # Test configuration sudo tail -f /var/log/apache2/access.log # View access logs

🔗 How They Work Together
🐳
Docker
Packages your app + Nginx (or Apache) into one portable container.
🦈
Nginx
Serves your files inside the Docker container. Fast and lightweight.
🦁
Apache
Alternative web server inside Docker. Great for PHP/WordPress.
📝 They Work Like This: 1. You write a Dockerfile (recipe) 2. Docker builds an image 3. Docker runs a container from that image 4. Inside the container, Nginx (or Apache) serves your website 5. You access it at http://localhost:8080

✅ A blueprint/recipe for creating containers

✅ Serving static files and reverse proxy

✅ docker build -t my-app .
Q4: What is the main difference between Nginx and Apache?
✅ Nginx is faster and lighter; Apache has more features and heavier
Q5: What is the command to start Nginx on Ubuntu?
✅ sudo systemctl start nginx
📋 Quick Reference Card
🐳
Docker Cheat Sheet

List: docker ps
Stop: docker stop name
Remove: docker rm name
🦈
Nginx Cheat Sheet

Stop: sudo systemctl stop nginx
Restart: sudo systemctl restart nginx
Test: sudo nginx -t
Reload: sudo nginx -s reload
🦁
Apache Cheat Sheet
Start: sudo systemctl start apache2
Stop: sudo systemctl stop apache2
Restart: sudo systemctl restart apache2
Test: sudo apache2ctl configtest
Enable: sudo a2ensite site.conf
🎯 Why This Matters

🔹 Docker = Run your apps anywhere, no "it works on my machine" problems

🔹 Nginx = Fast web server, great for serving files and handling many users

🔹 Apache = Reliable web server, great for PHP apps like WordPress

🔹 Together = Docker packages your app + Nginx (or Apache) into one portable box

🎮 Pro Tip: Nginx is the "speed" choice, Apache is the "features" choice. Both are awesome!
